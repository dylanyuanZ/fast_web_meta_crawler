# Section 引导指南

## Section 概览

| Section | 思考框架（AI 提供） | 内容（用户提供） |
|---------|-------------------|-----------------|
| **4.1 方案概览** | 模块划分、依赖关系、数据流、架构融合 | 用户描述整体设计 |
| **4.2.1 核心模块** | 职责划分、包设计、接口抽象 | 用户描述模块设计 |
| **4.2.2 接口设计** | 契约、兼容性、错误处理 | 用户描述接口 |
| **4.2.3 数据模型** | 字段、关系、存储格式 | 用户描述数据结构 |
| **4.2.4 并发模型** | goroutine 模型、channel 设计、同步策略 | 用户描述并发策略 |
| **4.2.5 错误处理** | 异常分类、恢复策略、降级方案 | 用户描述错误处理 |
| **4.3 核心逻辑** | 算法复杂度、边界情况、关键路径（**完成后调用 `bp-performance-optimization` Skill**） | 用户描述实现逻辑 |
| **4.4 方案优劣** | 优点、局限、风险、改进空间 | 用户总结优劣 |
| **5. 备选方案** | 为什么不选、什么情况下会重新考虑 | 用户整理备选方案 |
| **7. 测试计划** | 测试类型、覆盖范围、关键用例（**调用 `workflow-test-generation` Skill**） | 用户描述测试策略 |
| **8. 可观测性** | 日志、监控指标 | 用户描述可观测性 |

---

## 严格顺序：架构 → 组件 → 核心逻辑

**4.1 / 4.2 / 4.3 必须按由抽象到具体的顺序逐级完成，禁止跳级。**

| 规则 | 说明 |
|------|------|
| **4.1 未完成前，不讨论 4.2/4.3** | 架构方向未定，组件和实现细节的讨论没有锚点 |
| **4.2 未完成前，不讨论 4.3** | 接口未定，核心逻辑无从附着 |
| **不提前写入后续 section** | 即使用户在 4.1 讨论中提到了具体实现，也只记录到备忘，等进入对应 section 时再正式写入 spec.md |

---

## 内容粒度标准

| 维度 | 4.1 方案概览 | 4.2 组件设计 | 4.3 核心逻辑 |
|------|-------------|-------------|-------------|
| **抽象层次** | 模块边界、依赖方向、数据流走向 | 包/接口定义、职责划分、并发模型 | 算法伪码、调用点清单、具体改动说明 |
| **命名粒度** | 模块名、包名 | 接口名、struct 名 | 函数名、变量名 |

## 各 Section 引导模式

### 4.1 方案概览（最关键）

**引导框架**：
```
我们现在讨论 **4.1 方案概览**。

请描述你的整体设计思路，包括：
- 你打算怎么划分模块/包？
- 模块之间的依赖关系是什么？
- 数据流是怎样的？
- 和现有代码的关系？新增还是修改？
```

**用户描述后，AI 质疑**：
- "你把 X 和 Y 放在一起，它们的职责是否耦合过紧？"
- "A 依赖 B，B 又依赖 A，这是循环依赖，你打算怎么解决？"
- "这个设计改动了 N 个文件，有没有更小范围的方案？"

### 4.2 组件设计

按子节逐个讨论。每个子节：

```
AI："我们现在讨论 **4.2.1 核心模块设计**。

请描述你打算设计哪些模块/包，它们的职责分别是什么？"
```

### 4.3 核心逻辑 + 性能优化

**引导框架**：
```
AI："我们现在讨论 **4.3 核心逻辑**。

请描述核心算法和实现逻辑：
- 关键路径是什么？
- 算法复杂度如何？
- 有哪些边界情况需要处理？"
```

**核心逻辑设计完成后，加载 `bp-performance-optimization` Skill 从性能角度审视设计。**

### 6. 业界调研（AI 辅助）

这是唯一一个 AI 可以主动提供信息的 section：
```
AI："第 6 节业界调研，我来搜索一下相关的业界方案。"
```

### 7. 测试计划（调用 Skill）

```
AI："我们现在讨论 **7. 测试计划**。

我会加载 `workflow-test-generation` Skill 来帮助设计更完备的测试计划。"
```

## 判断 Section 是否适用

```
AI："4.2.3 数据模型这个 section，你的需求涉及新的数据结构吗？不涉及可以跳过。"
```

**用户选择跳过时**：在 spec.md 该 section 标注 `N/A - 本需求不适用`。
