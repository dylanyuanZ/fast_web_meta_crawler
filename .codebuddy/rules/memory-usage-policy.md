# Memory 使用策略

## 核心原则

**项目知识必须跟随项目仓库走**（`.codebuddy/rules/` + `skills/`），不与个人账号绑定。memory 仅允许存储**纯个人偏好**（语言、格式风格等不涉及项目的个人习惯）。

## 强制规则

1. **禁止仅存 memory**：项目相关知识（配置、流程、组件、经验教训）**必须写入 rules/skills 文件**
2. **减少 memory 使用**：只在内容纯属个人偏好时才使用
3. **及时清理**：memory 中已沉淀到 rules/skills 的内容应主动清理
4. **沉淀优先**：被用户纠正后调用 `self-refinement` skill 写入 rules/skills，而非存入 memory
