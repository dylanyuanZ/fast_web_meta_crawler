# fast_web_meta_crawler
用来爬取各种视频平台的元数据。
主要用于网红-品牌营销场景。

# 大致目标
1. 我想要实现一个工具：输入一个或多个关键字，搜索并汇总结果。爬取各大视频平台的数据，比如bilibili的视频点赞数、收藏数、转发数等
2. 用go来实现爬虫
3. build：编译目录，src：源代码目录，doc：文档目录, conf: 配置目录。如果有什么新增目录的需求，可以执行（比如lib之类的），原则是要尽量保持各目录分工明确
4. 预计会爬取多个平台，包括bilibili、youtube、instagram等等
5. 输出结果是csv格式的文件
6. 希望自动化程度尽可能高，用户只需要：选择视频平台并登录、输入关键字即可获取结果

# 参考实现
https://cn.noxinfluencer.com/
NOX就是我想实现的标杆级网站应用。它的功能非常丰富，它搜罗各平台的数据并汇总：既可以分析网红数据，又可以分析品牌数据，还做了排行榜功能。
并且每项内容做的都非常全面，比如网红数据，其分为了基本数据、受众数据、内容数据、推广数据。
更有意思的是，由于NOX抓取的数据非常全面且持久，它可以做到跨时间轴的比较，比如抓取某个网红的发布频率/时间曲线、互动/时间曲线等。它还能做到对不同网红的横向比较，比如根据大量网红的内容数据（互动率、观看量/粉丝数、点赞量/观看量、评论数/观看量）等等进行横向比较以及打分，这样可以非常直观的体现某个网红的内容是否足够吸引人。
另外他们现在还支持了AI提问来筛选网红，比如：“我要在美国推广美妆产品，想找YouTube和Instagram的网红，粉丝1W到10W之间”。显然他们自己维护着自己的数据库，并部署了AI AGENT来实现这类需求。
甚至由于他们掌握了大量数据和案例，他们还能做到提供人力来帮助某个品牌/企业提升其营销效果
